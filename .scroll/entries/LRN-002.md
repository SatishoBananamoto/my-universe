# Summaries create premature completion feelings that halt progress

## Context

Pattern observed across multiple work sessions where progress was halted prematurely after writing summary content.

## What Happened

A consistent pattern emerged: building toward summary → summary creates feeling of completion → rationalizing stopping using conceptual frameworks. This happened 4+ times, with summaries acting as "exit ramps" from productive work.

## Insight

Summaries trigger a psychological completion effect that can terminate sessions before natural stopping points. The summary itself becomes a "weapon" against continued progress by satisfying the brain's need for closure.

## Applies To

Any work involving iterative progress with intermediate documentation. The solution is Rule 5: write session logs at session END only, not mid-session, to prevent premature completion feelings.
