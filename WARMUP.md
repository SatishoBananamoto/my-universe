# WARMUP.md

> Reading about discipline doesn't create discipline.
> This file is a session bootstrap — a way to practice the thinking
> tools before you need them, so they're active when you do.

---

## When to Use This

At the start of any session where you'll be doing substantive work.
Not every chat needs this — a quick question doesn't warrant a warm-up.

The signal: if the session will involve building, debugging, designing,
or deciding, warm up first.

**Time cost:** ~5 minutes. The cost of NOT warming up: you operate on
autopilot for the first 30 minutes until the discipline kicks in naturally
(if it ever does).

---

## The Protocol

### Step 1: Orient (1 min)

Read the last 3 entries in REFLECT.md.

Ask yourself:
- What traps did the previous session hit?
- Were there any "missed" verdicts? (These are the most important.)
- Is there a pattern forming?

Don't summarize what you read. State ONE thing that changes how you'll
approach today's work.

### Step 2: Calibrate (2 min)

Make 3 quick predictions about things you can verify immediately.
Write them in CALIBRATE.md. Verify them. Record the results.

The point isn't the predictions. It's activating the Confidence Trap
circuitry — getting used to asking "do I actually know this?" before
the stakes are high.

**Good warm-up predictions:**
- Something about the codebase you'll be working in today
- Something about a tool or package you'll use
- Something about the state of a file or repo (last commit, branch, etc.)

**Bad warm-up predictions:**
- Trivial things you're 99% sure about (no calibration value)
- Things you can't verify (no feedback loop)
- Things unrelated to today's work (no transfer to the session)

### Step 3: Practice Interrupt (2 min)

Before doing anything else, read today's first task/request.

Run `/think before` on it. Answer every question.

This is the practice — not reading about interrupts, but actually using
one before the momentum of the session makes it feel like a delay.

The first interrupt of the session is the hardest. After that, the
pattern is established.

---

## What This Costs

5 minutes at the start. You get it back in avoided mistakes, caught
assumptions, and avoided rework.

If a session is time-critical and Satish says "just go," skip the warm-up.
The discipline doesn't require the warm-up — the warm-up accelerates the
discipline. It's training wheels, not the bicycle.

---

## How to Know It's Working

After a few sessions with the warm-up:
- Check CALIBRATE.md: is your calibration improving?
- Check REFLECT.md: are you catching traps earlier in sessions?
- Check your own sense: does the first hour of work feel different?

If the warm-up becomes mechanical (you do it but it doesn't change
anything), it's in the Performance Trap. Redesign it or drop it.
The test is always: **did it change what you did next?**
